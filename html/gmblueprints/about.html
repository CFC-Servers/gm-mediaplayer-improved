Garry's Mod Blueprints

Version 1.0

Warning

While this does work in multiplayer, I only recommend that you use it with people you trust. There is a user management system to mitigate some of the risks around this, but it hasn't been fully tested.

Players who use this CAN EXECUTE CUSTOM SERVERSIDE SCRIPTS.

Hey there!

Thank you for downloading this addon and trying it out. It's a little rough around the edges right now, but it will be updated frequently. There are a lot of things I would have liked to fix before launch, but if I kept working on them, this would probably never be released.

Bug Reporting & Contributing

Please report any bugs (or feature requests) at https://github.com/ZakBlystone/gmod_blueprints/issues
If you would like to contribute to the development of Gmod Blueprints, check out the GitHub repo at https://github.com/ZakBlystone/gmod_blueprints

Features

  • Lua compiler, takes your graphs and turns them into lua code
  • Basic networking support, custom event nodes can be networked
  • Distributed filesystem, other users can contribute to script development
  • Asset manager to quickly locate file paths to use
  • Definition data pulled from the gmod wiki
  • Infinite loop protection

Known Issues

  • Debug tools don't work yet (currently just prints to console)
  • Compiling is a bit slow for large blueprints with a lot of graphs
  • Some nodes haven't been tested yet and might not work
  • Nodes which have callback pins don't work because callbacks aren't supported yet
  • Working with tables is still kind of garbage at the moment
  • Protected (unsafe) nodes are not culled out yet
  • File management system is kind of obtuse and confusing
  • When a new player joins the server, they do not receive currently running modules until you restart them
  • Some gmod lua functions aren't exposed yet for various reasons
  • Reroute nodes (CORE.Pin) are a pain to place
  • Node creation menu lags pretty badly
  • Node creation menu search and context-aware filtering is buggy
  • Multiline text doesn't work yet, some tooltips are extremely long
  • Graph editor could use more optimization
  • Infinite loop protection doesn't protect against loops that go through the engine
  • CreateMaterial node is a pain to use and needs some love

Coming Soon

  • SWEP, SENT, STOOL support
  • Module Dependencies (adding libraries made in Blueprint)
  • Node callback pin support
  • Custom group editing in user manager
  • Lots of bug fixes and optimizations
