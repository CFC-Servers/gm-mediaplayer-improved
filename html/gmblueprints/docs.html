Getting Started

Creating a module

  1. Hold C to open the context menu and click on the Blueprint Editor icon
    You can bind a shortcut to this in console using bind <key> bp_open_editor
  2. Select the Files tab and click New Module
  3. Give the module a unique name

Your first blueprint

Adding Nodes

  1. Right-click anywhere in the graph editor to bring up the creation menu
  2. In the search-bar type player. and select player.GetAll, you should see the node appear in the graph
  3. Repeat these steps and place CORE.ForEach, Player:ChatPrint, and CORE.ServerOnly your graph should look something like this:
  4. On the Player:ChatPrint node, Left-click the small box next to the
    message     pin, a dialog will show up and you can enter any message you'd like, press Enter to complete.

Connecting Nodes

  1. To connect nodes, Left-click the pin you'd like to connect, and drag your mouse over to the pin you want to connect it to.
    Connections are bi-directional it doesn't matter what order you connect the pins.
  2. If you release your left-mouse button on the graph,
    the creation menu will appear and will be pre-filtered to show only nodes that accept the pin type you started dragging from.
  3. To close this menu, just click anywhere on the graph
  4. To disconnect a pin, hold Ctrl and left-click the pin you want to disconnect, drag the connection off of the pin and release

Finishing Up and Running

  1. Connect the nodes as shown
  2. Click Send to server and close the editor, you should now see your message printed in chat for all connected players.

Making Adjustments

  1. Left-click on the graph and drag to select multiple nodes
  2. Press Delete to delete the selected nodes

Managing Files

This is the file manager, here you can upload / download modules, toggle which modules are currently running, and delete files.
The left panel shows all the files you have locally on your machine.
The right panel shows all files that have been uploaded to the server.
Double-click on a local file to open it.

Right click a file to access options

When a file is green, it means that the file is locked by you. Nobody else can edit that file on the server while you have it locked.
When a file is red, it means that the file is locked by someone else and you cannot edit it.

Managing Users

This is the user manager, here you can assign permissions to users to control who can do what in the blueprint system.
Simply check the box next to which role you want to assign to a user.

Currently you can only assign a user between 4 pre-designed groups. Group creation / editing coming soon.

Admin: Can do anything.
Auditor: Can toggle blueprints. (soon to be able to view any server blueprint in read-only mode).
Trusted: Can upload blueprints to the server, can run blueprints locally, and can toggle which blueprints are running. They are also allowed to use protected nodes
Newbie: Can only upload blueprints to the server. Their blueprint will not run until someone toggles it for them.

By default, any new player who joins isn't a member of any of these groups, and cannot do anything with blueprints.
Group / user information is stored persistently on the server.
